--- layout: post title: "Software as a Service" date: 2011-05-27 comments: false --- Algunas de las necesidades claves para adoptar el servicio de Software as Service según artículo de Paul Avenant publicado en Software Guru (http://www.sg.com.mx/content/view/1136/2/) son las siguientes:


  • Falta de tiempo para desarrollo de nuevas soluciones y para su administración.
  • Necesidad de evitar costos de soporte, software e infraestructura.
  • Seguridad de la información sin la capacidad para proveer esta característica.
  • Requisitos variables de servicios o crecimiento imprevisible.
